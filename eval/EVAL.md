# Evaluation Guide

This folder contains tools and guidelines for evaluating videos generated by Image-to-Video (I2V) models on tasks from this pipeline's dataset.

---

## Overview

After you download and convert a dataset with this pipeline, the next step is:

1. Feed `first_frame.png` + `prompt.txt` to a video generation model
2. The model produces a video
3. **Evaluate** that video against the ground truth

This folder helps with step 3.

---

## What's Inside

| File | Purpose |
|---|---|
| `verify.py` | Automated rule-based evaluation script |
| `EVAL.md` | This document |

You can extend this folder with whatever fits your dataset — human evaluation rubrics, comparison apps, Jupyter notebooks, Streamlit dashboards, etc.

---

## Automated Evaluation

### Quick Start

```bash
# Install eval dependencies
pip install -r eval/requirements.txt

# Evaluate a single video against one GT sample
python eval/verify.py \
    --video path/to/generated_video.mp4 \
    --gt-dir data/questions/videothinkbench_task/vtb_test_00000/

# Batch evaluate all videos against GT
python eval/verify.py \
    --videos-dir path/to/model_outputs/ \
    --gt-dir data/questions/

# Save results as JSON
python eval/verify.py \
    --video path/to/generated_video.mp4 \
    --gt-dir data/questions/videothinkbench_task/vtb_test_00000/ \
    --output results.json
```

### Output

```
Task: vtb_test_00000
  first_frame_consistency:  0.95  (weight: 0.15)
  final_frame_accuracy:     0.72  (weight: 0.35)
  temporal_smoothness:      0.88  (weight: 0.15)
  visual_quality:           0.81  (weight: 0.10)
  task_specific:            0.65  (weight: 0.25)
  ─────────────────────────────────────────────
  Overall:                  0.77
```

### Evaluation Dimensions

| Dimension | Weight | What It Measures |
|---|---|---|
| `first_frame_consistency` | 15% | Does the video's first frame match the GT input image? (SSIM) |
| `final_frame_accuracy` | 35% | Does the last frame match the expected final state? (SSIM + PSNR) |
| `temporal_smoothness` | 15% | Are transitions smooth, no flickering or jumps? |
| `visual_quality` | 10% | Sharpness and noise levels |
| `task_specific` | 25% | Custom rule-based logic for this task |

These weights and dimensions follow the [VBVR-EvalKit](https://github.com/Video-Reason/VBVR-EvalKit) convention.

## Customizing the Evaluator

Open `eval/verify.py` and modify the `TaskEvaluator` class:

1. **Define your sub-criteria** in `TASK_WEIGHTS`
2. **Implement detection/measurement** in helper methods
3. **Score each criterion** in `evaluate_task_specific()`

The included example uses a generic frame-similarity evaluator suitable for most pipeline datasets. Replace it with task-specific logic if your dataset requires it.

---

## Human Evaluation

For datasets where automated metrics may not capture all quality aspects, consider adding a `human_eval.md` file with:

- What to look for when judging generated videos
- A scoring rubric (e.g., 1–5 scale per criterion)
- Examples of good and bad generations
- Side-by-side comparison instructions

## Directory Structure Expected

**Ground truth** (produced by the pipeline):
```
data/questions/{domain}_task/{task_id}/
├── first_frame.png
├── final_frame.png
├── prompt.txt
├── first_video.mp4   (optional)
├── last_video.mp4    (optional)
├── ground_truth.mp4  (optional)
└── metadata.json     (optional)
```

**Model outputs** (for batch evaluation):
```
model_outputs/
├── {task_id}.mp4
└── ...
```
